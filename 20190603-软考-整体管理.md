#### 项目整体管理

**1. 启动过程组-制定项目章程：**

> 项目章程是正式批准的文件，授权项目经理在项目活动中动用组织的资源，所以项目经理在任何时候都应该在规划开始之前进行委派，最好是在制定项目章程的时候。

**项目章程应当包括以下内容：** 

> * 项目目的或批准项目的原因。
> * 可测量的项目目标和相关成功的标准。
> * 项目的总体要求
> * 概括性的项目描述
> * 项目的主要风险
> * 总体里程碑进度计划
> * 总体预算
> * 项目审批要求：用什么标准来评价项目成功，由谁对项目成功进行评价，由谁来签署项目结束
> * 委派项目经理及职责和职权
> * 发起人或其他批准项目章程人员的人员的姓名和职位

**4.2.1 制定项目章程**

> 制定项目章程是制定一份正式批准项目或阶段的文件。在项目中，应该尽早确认并任命项目经理，由于项目章程将授权项目尽力在以后的开发活动中使用组织资源，故项目经理也应该参与制定项目章程。

**制定项目章程的依据：**

1. 协议：

   > 协议定义了项目启动的初衷。协议的形式有：合同、谅解备忘录、服务品质合同、协议书、意向书、口头协议、电子邮件或其他书面协议。通常为外部客户做项目时，就用合同。

2. 项目工作说明书：

   > 工作说明书是对项目提供的产品或服务的叙述性说明。
   >
   > 项目说明说明书指明下列事项之一即可：
   >
   > > * 业务需求：组织的业务需求可能基于培训需求、市场需求、技术进步、法律要求或政府标准。
   > > * 产品范围说明书： 是项目创造的产品或服务要求和特征文件。
   
3. 商业论证：

   > 商业论证或类似文件能从商业角度提供必要的信息，决定项目是否值得投资。
   >
   > 可以从以下一个或几个原因而编制商业论证：
   >
   > > * 市场需求
   > > * 组织需要
   > > * 客户要求
   > > * 技术进步
   > > * 法律要求
   > > * 生态影响
   > > * 社会需要

4. 事业环境因素：

   > 在制定项目章程时，任何一种以及所有存在于项目周围并对项目成功有影响的组织事业环境因素与制度都必须加以考虑(**即：项目的限制因素**)

5. 组织过程资产：

   > 在制定项目章程及以后的项目文件时，任何一种以及所有影响项目成功的的资产都可以作为组织过程资产。

**4.2.5 项目启动会议：**

> * 项目启动会议由项目经理召开，他是启动项目的常用方式。召开项目启动会议的目的在于使项目的主要利益相关者	明确项目的目标、范围、需求、背景和各自的职责和权限。
>
> * **项目启动会议的五个方面的工作：**
>
>   > * 确定会议目标
>   > * 做好会议前的准备工作
>   > * 明确并通知参加会议的人员
>   > * 明确会议的主要议题
>   > * 做好记录

**项目的目标和项目的目的：**

> * 项目的目的： 目的就是想要达到的结果，如盈利等
> * 项目的目标：它的用途之一是衡量项目目的是否达到，结果是否符合事先说明的需要和要求；也是对项目目的的量化
>
> **目的可能只有一个，但是目标可以有多个**

**4.5.1 监控项目工作：**

**1.监控项目工作的输入：**

|     输入     |
| :----------: |
| 项目管理计划 |
|   进度预测   |
|   成本预测   |
|  确认的变更  |
| 工作绩效信息 |
| 事业环境因素 |
| 组织过程资产 |

**1.1 项目管理计划：**

> 监控项目工作包括查看项目的各个放面。项目管理计划中的子计划是控制项目的依据。包括但不限于：范围、进度、成本和质量管理计划，过程改进计划，人力资源管理计划，风险、采购管理计划，干系人管理计划和三个基准(范围、进度和成本基准)

**1.2 进度预测：**

> 基于项目实际进展和进度基准的比较计算出进度预测，即完工尚需时间估算，通常表示为进度偏差和进度绩效指数。

**1.3 成本预测：**

> 基于项目的实际进展和成本基准的比较计算出成本预测，即完工尚需估算，通常表示为成本偏差和成本绩效指数。通过比较完工估算和完工预算，可以看出项目是否在可以容忍的范围内，是否需要提出变更请求。

**1.4 确认的变更：**

> 批准的变更是实施整体变更控制过程的输出。需要对它们执行情况进行确认，以保证它们都得到正确的落实。

**1.5 工作绩效信息：**

> 工作绩效信息是从各个控制过程中收集并结合相关背景和领域关系，进行整合而得到的绩效数据。工作绩效数据转化为工作绩效信息，工作绩效信息通过沟通进行传递。绩效信息包括可交付成果的状态、变更请求的落实情况和预测的完工尚需估算。

**变更请求：**

> 所有的监控过程和执行过程都会输出"变更请求"。并更请求包括纠正措施，预防措施和缺陷补救。预防措施和纠正措施不会影响项目基准，而只影响相对于基准的项目绩效
